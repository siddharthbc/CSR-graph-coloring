Cerebras Wafer-Scale Engine Hardware Overview

The WSE-2 has 850,000 cores arranged in a 2D grid. Each Processing Element (PE) has its own compute, 48KB memory, and a router connected to 4 cardinal neighbors + its own RAMP.

Each PE is an independent processor with local SRAM. PEs communicate via fabric colors — named wavelet channels routed through the 2D mesh. The host (Python, SdkRuntime) orchestrates data transfer and kernel launches.

Graph Partitioning onto PEs Host → memcpy_h2d

The host partitions the conflict graph's CSR arrays across PEs. Each PE gets a contiguous slice of vertices and their adjacency data. Edges crossing PE boundaries are "boundary edges" — they require inter-PE communication.

Speculative Parallel Coloring pe_program.csl → color_step()

Unlike the sequential bucket algorithm, on WSE each PE simultaneously picks tentative colors for all its uncolored vertices. Conflicts are detected and resolved in a subsequent phase.

Wavelet Communication Fabric DSDs / Routes

After tentative coloring, each PE sends its boundary vertex colors to neighbors via fabric colors. The router at each PE has 5 ports: RAMP (to local CE), NORTH, SOUTH, EAST, WEST. Colors define wavelet channels.

Step 1 / 4
PEs use fabout_dsd and fabin_dsd to send/receive wavelets along routed fabric colors.

Conflict Detection & Resolution Priority Rule

After receiving neighbor colors, each PE checks boundary edges for conflicts. Resolution: for edge (u,v) with same color, the vertex with higher ID yields — resets to uncolored and retries next round.

Step 1 / 4
Priority-based resolution guarantees progress: each round, at least one endpoint of each conflicting edge keeps its color.

Full CSL File Architecture 3-file structure

A Cerebras CSL program consists of a layout file (PE grid setup), a PE program (kernel code), and a host Python driver.

Compile: cslc layout.csl --fabric-dims=11,3 --fabric-offsets=4,1 --params=... --memcpy --channels=1 -o out
Run: cs_python run.py --name out (simulator)  |  cs_python run.py --name out --cmaddr IP:PORT (real hardware)
