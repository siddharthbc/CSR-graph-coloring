Every Pauli string is built from four basic 2×2 matrices: I, X, Y, Z. Before understanding commutation of strings, let's see how these individual matrices commute with each other.
I
Identity
[ 1 0 ]
[ 0 1 ]
Does nothing.
Commutes with everything.
X
Pauli-X (bit flip)
[ 0 1 ]
[ 1 0 ]
Flips |0⟩ ↔ |1⟩
Like a NOT gate.
Y
Pauli-Y (flip + phase)
[ 0 -i ]
[ i 0 ]
Flips with a phase.
Combines X and Z.
Z
Pauli-Z (phase flip)
[ 1 0 ]
[ 0 -1 ]
Flips the phase of |1⟩.
Leaves |0⟩ alone.
Key fact: Any two distinct non-identity Pauli matrices anticommute. That means X·Y = −Y·X, and Y·Z = −Z·Y, and X·Z = −Z·X. Same matrices commute: X·X = X·X. And I commutes with everything.
Single-matrix commutation table:
IXYZ
I—CCC
XC—AA
YCA—A
ZCAA—
C = commute (same or involves I)  |  A = anticommute (different non-identity)
The rule for single matrices is simple: Two non-identity Pauli matrices anticommute if and only if they are different. X vs Y → anticommute. X vs X → commute. Anything vs I → commute. This is the building block for checking entire strings.
A Pauli string is N matrices side by side, one per qubit. To check if two strings commute or anticommute, compare them position by position and count the mismatches.
The algorithm:
1. Compare position by position
2. At each position: if BOTH characters are non-I AND different → that's a mismatch
3. Count total mismatches
4. Odd count → anticommute  |  Even count (including 0) → commute
Example 1: XXYY vs XXZZ
Example 2: XXYY vs YYXX
Example 3: XYXY vs YXYX
Why only odd/even matters: Each position-wise anticommutation contributes a factor of −1. Multiply all factors together: an odd number of −1s gives −1 overall (anticommute), an even number gives +1 (commute). It's just parity.

The key equation (Eq. 2 from the paper):

For m Pauli strings to be combined, we need: Σᵢ,ⱼ p*ᵢpⱼPᵢPⱼ = 0

This sum equals zero ONLY when every pair anticommutes, because:
• If PᵢPⱼ = −PⱼPᵢ (anticommute), then p*ᵢpⱼPᵢPⱼ + p*ⱼpᵢPⱼPᵢ = 0 — the terms cancel!
• If PᵢPⱼ = +PⱼPᵢ (commute), then p*ᵢpⱼPᵢPⱼ + p*ⱼpᵢPⱼPᵢ ≠ 0 — they add up instead of canceling

So mutual anticommutation is the mathematical condition that makes grouping valid.
Connection to graph coloring:

• Anticommute → edge in original graph G → can be in the same clique → same group ✓
• Commute → edge in complement graph G' → must have different colors → different groups ✓

Valid coloring of G' ensures every group only contains mutually anticommuting strings. That's the entire bridge from quantum physics to graph theory.
Let's see all pairs from a set of Pauli strings, which ones commute, which anticommute, and what that means for grouping. This is exactly what Picasso computes on-the-fly.
Anticommuting pairs (edges in G):
These strings CAN be grouped together. In the graph, they're connected. A clique of mutually anticommuting strings = one unitary = one measurement round.
Commuting pairs (edges in G'):
These strings MUST be in different groups. In the complement graph, they're connected. Same color = violation. Picasso colors G' to prevent this.
The complete picture:

1. Start with Pauli strings from molecular Hamiltonian
2. Each pair either commutes or anticommutes (determined by position-wise mismatch parity)
3. Anticommuting pairs → edges in original graph G (compatible for grouping)
4. Commuting pairs → edges in complement graph G' (must separate)
5. Color G' validly → same-color vertices form cliques in G → each clique is one unitary
6. Picasso does step 5 memory-efficiently using O(n log³n) conflict subgraphs

Everything traces back to this one question: do these two strings commute or anticommute?
