Link 5: We need to group mutually anticommuting Pauli strings. This is a graph problem: make each string a vertex, connect anticommuting pairs with edges, then find cliques (groups where every pair is connected).

Link 6: Finding cliques in graph G is the same as coloring its complement graph G'. The paper uses this equivalence because graph coloring has better algorithms than clique partitioning.
Step 0 of 5
What you're about to see
We'll see the same 8 strings in two graphs side by side: the original graph G (anticommuting edges) and its complement G' (commuting edges). Then we'll see how cliques in G = colors in G'. Click ▶ to start.
Link 7: To color G', existing algorithms need the entire graph stored in memory. For real molecules, these graphs have trillions of edges requiring terabytes of storage. GPU memory is only 40 GB. This is the wall.
Step 0 of 5
What you're about to see
We'll scale up from our toy example to real molecules and see exactly where existing algorithms hit the memory wall. Click ▶ to start.
Link 8: Picasso's insight: don't store the full graph. Store only the Pauli strings. Each iteration, build a tiny conflict subgraph, color what you can, and repeat.
