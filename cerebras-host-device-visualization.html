Compile CSL Program (cslc)

4Upload

Host → Device Memory Transfers H2D

For each PE, upload via memcpy_h2d:

  • csr_offsets[] — CSR row offsets (padded to max_local_verts+1)
  • csr_adj[] — CSR adjacency (padded to max_local_edges)
  • colors[] — initialized to -1 (uncolored)
  • local_num_verts — how many vertices this PE owns
  • global_vertex_ids[] — GID of each local vertex
  • boundary_local_idx[] — local indices of boundary vertices
  • boundary_neighbor_gid[] — GIDs of cross-PE neighbors
  • num_boundary — count of boundary edges
  • expected_recv_east / expected_recv_west — wavelet counts for barrier
picasso/cerebras_host.py :: run_coloring()
▼
5Color!

Autonomous On-Fabric Coloring DEVICE

Host calls runner.launch('start_coloring') once, then blocks.

Each PE runs a BSP loop for max_rounds iterations autonomously via task chaining — no host interaction.

6Readback

Device → Host Memory Transfers D2H

  • For each PE, memcpy_d2h reads colors[] array
  • Assembles global color vector: all_colors[start_v + i] = buf[i]
  • Calls runner.stop()

7Validate

Validate & Output

  • Check no two adjacent vertices share the same color
  • Check no uncolored vertices remain (color == -1)
  • Compare against golden reference files
  • Print color assignment as JSON
picasso/run_csl_tests.py :: validate_coloring()
Key Design Insight: The host only touches the device twice — once to upload data (Step 4) and once to read back results (Step 6). The entire multi-round coloring algorithm (Step 5) runs autonomously on-fabric with zero host involvement. This is what makes the Cerebras implementation fast: no PCIe round-trips in the critical path.

Wavelet Packing (u32)

Bits 31–16: Vertex GID
Bits 15–0: color + 1
Single 32-bit wavelet carries one boundary vertex's identity and tentative color

Summary: Host vs. Device Responsibilities

Files: cerebras_host.py (host) · pe_program.csl (device) · layout.csl (fabric)

🖥️ Host (CPU)

  • Load Pauli JSON input
  • Build commutativity conflict graph
  • Convert to CSR format
  • Partition graph across PEs (contiguous chunks)
  • Compute boundary metadata + expected recv counts
  • Compile CSL program with cslc
  • Upload per-PE data via memcpy_h2d
  • Launch start_coloring (single call, then wait)
  • Read back colors via memcpy_d2h
  • Validate correctness + compare golden
cerebras_host.py · run_csl_tests.py

⚡ Device (Each PE)

  • Speculate: greedy min-color for uncolored vertices
  • Local conflict resolution: higher GID yields
  • Pack boundary wavelets: (GID << 16) | (color+1)
  • Async DMA east/west send buffers
  • Receive wavelets via data tasks
  • Implicit barrier via wavelet counting
  • Detect boundary conflicts from remote colors
  • Yield on conflict (revert to uncolored)
  • Task-chain to next round (no host involvement)
  • Exit after max_rounds → unblock host
