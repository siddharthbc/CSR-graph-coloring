Full Pipeline Overview

The Picasso algorithm colors Pauli strings so that commuting strings can share a group (color). It works in three phases, optionally recursing on uncolored vertices.

Pauli String Commutativity pauli.py → is_an_edge()

Two Pauli strings anti-commute iff the number of qubit positions where both are non-I and different is odd. Anti-commuting pairs form edges in the commutativity graph; they cannot share a group.

Step 1 / 4
We have 4 Pauli strings. Let's check which pairs commute vs anti-commute.

Palette Assignment palette_color.py → _assign_list_color()

Each vertex gets T = ⌊α · ln(n)⌋ unique random colors drawn from [0, palSize-1] using MT19937. Colors are sorted. This is the "list" in "list coloring".

Step 1 / 5
Each vertex receives a random subset of colors from the global palette.

Conflict Graph Construction graph_builder.py → build_conflict_graph()

For each commuting pair (u,v), if their color lists share a common element, we add a conflict edge. The edge list (COO) is converted to CSR via scipy.sparse.

Step 1 / 5
Scan all commuting pairs and check for color-list overlap.

Bucket-Based Greedy Coloring palette_color.py → conf_color_greedy()

Vertices are placed in buckets by list size (smallest first). Each round: pick a random vertex from the smallest non-empty bucket, assign a tentative color, then propagate removals to neighbors via the CSR conflict graph (_fix_buckets).

Recursion & Naive Fallback pipeline.py → run()

Invalid vertices (exhausted color lists) get a fresh palette in a higher color range and re-enter the pipeline. When few remain (≤ n_inv), a simple greedy fallback finishes the job.
